## Goal

Set up and configure a TP-Link Archer T2U Plus (RTL8821AU) Wi-Fi USB adapter on a Raspberry Pi running Raspberry Pi OS Lite 64-bit for a Wi-Fi CPE project that analyzes Wi-Fi interference, detects networks, and provides recommendations.

## Instructions

- User has a Raspberry Pi (headless, no desktop) connected via LAN cable for SSH management
- The TP-Link Archer T2U Plus adapter uses RTL8821AU chipset - requires custom Linux driver for monitor mode
- Project is called "wa-ai-project" / "WA-Repo" on GitHub
- User wants live analysis, NOT demo mode
- Dashboard should auto-start on boot with login protection
- Credentials: admin / Logmein@1
- Pi IP: 192.168.0.150 (may vary)

## Discoveries

- RTL8821AU driver doesn't report channel info in monitor mode (shows as -1)
- Using channel-by-channel scanning with airodump-ng resolves the limitation
- Monitor mode and connected Wi-Fi are mutually exclusive
- wlan0 = built-in Pi WiFi, wlan1 = TP-Link adapter
- Wi-Fi analyzer can detect: networks, connected devices, devices searching for Wi-Fi, channel congestion
- Manufacturer lookup via OUI database identifies device brands (Apple, Samsung, HP, Dell, etc.)
- Hidden SSIDs cannot be directly revealed, but devices connected to them can be tracked
